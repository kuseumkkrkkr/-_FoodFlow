# 첫 MVP 시작 케이싱

## 결론

첫 MVP는 모든 식품 OEM/ODM을 다루지 않는다. 가장 빠른 시작 범위는 `저당·고단백 건강간식/분말·스틱`을 1순위로 잡고, `소스 OEM`을 B2B 반복 발주 보조 케이스로 붙이는 방식이다.

## 왜 이렇게 줄이는가

| 제외/후순위 | 이유 |
|---|---|
| 전체 HMR | 냉장·냉동, 소비기한, 물류, 샘플 검증 난도가 높다 |
| 수산/축산 HMR | 원료 수급, 위생, 보관, 콜드체인 조건이 무겁다 |
| 건강기능식품 전체 | 기능성 표시와 인허가 검토가 필요해 초반 범위가 커진다 |
| 모든 B2B/B2C 동시 대응 | 입력 폼과 공장 필터가 너무 넓어져 매칭 품질 검증이 늦어진다 |

## MVP 1차 타겟

### Case 001. 저당·고단백 건강간식

- 사용자: 예비 창업자, D2C 브랜드, 공동구매 운영자
- 제품 예시: 저당 곡물스낵, 고단백 쿠키, 식이섬유 바, 단백질 그래놀라
- 필요한 공장 조건: 곡물/스낵 제조, 배합, 성형, 굽기/건조, 개별 포장, HACCP, 알레르기 관리
- 바이브 쿠킹 산출물: 레시피 초안, 원재료 BOM, 대체당/단백질원 후보, 목표 원가, 샘플 요청서
- 성공 기준: 공장 3곳 이상이 같은 입력으로 견적 가능 여부를 판단할 수 있어야 한다

### Case 002. 분말·스틱 건강식품

- 사용자: 건강식품 브랜드, 인플루언서 커머스, 구독형 식단 서비스
- 제품 예시: 단백질 쉐이크, 식이섬유 분말, 저당 라떼 파우더, 기능성 콘셉트 분말
- 필요한 공장 조건: 분말 배합, 스틱포/파우치 포장, 원료 소싱, GMP/HACCP, 라벨 검수
- 바이브 쿠킹 산출물: 배합 방향, 기능성 표현 금지어 체크, 원료 역할 분류, 스틱 단위 원가 가정
- 성공 기준: 소량 MOQ 가능 업체와 대량 B2B 업체를 분리해 추천할 수 있어야 한다

### Case 003. 프랜차이즈·브랜드 소스 OEM

- 사용자: 프랜차이즈, 식당 브랜드, PB/편의점 상품기획자
- 제품 예시: 저당 소스, 매운맛 소스, 비건 드레싱, 덮밥 소스, 한식 양념장
- 필요한 공장 조건: 소스 R&D, 살균/충진, 파우치/병 포장, B2B 납품, HACCP
- 바이브 쿠킹 산출물: 맛 프로파일, 원재료 BOM, 점도/가열 안정성 리스크, 포장 단위, 샘플 비교표
- 성공 기준: 같은 소스 콘셉트에서 OEM형과 ODM형 문의를 분리할 수 있어야 한다

## MVP에서 받는 입력

| 필드 | 예시 | DB 필터 연결 |
|---|---|---|
| sales_type | B2B, B2C | 발주 흐름 |
| product_case | 건강간식, 분말스틱, 소스 | 공장 카테고리 |
| target_claim | 저당, 고단백, 식이섬유, 비건 | 원료/표시 조건 |
| texture_or_format | 바삭함, 분말, 액상, 소스 | 공정 조건 |
| target_qty | 1,000개, 5,000포, 1톤 | MOQ |
| package_type | 개별포장, 스틱, 파우치, 병 | 포장 설비 |
| channel | D2C, 공동구매, 프랜차이즈, PB | 납품/물류 |
| must_have | HACCP, GMP, 할랄, 비건 | 인증 |

## 개발 우선순위

1. `Factory` 테이블: 공장명, 카테고리, 생산 가능 품목, 인증, 위치, 출처, 검증 상태
2. `ProductCase` 테이블: 건강간식, 분말스틱, 소스 3개만 시작
3. `FactoryCapability` 테이블: 공정, 포장, 원료, MOQ 신호를 단순 필드로 관리
4. `VibeCookingSpec` 테이블: 사용자 입력을 BOM과 공정 조건으로 구조화
5. `RegulatoryScreening` 테이블: 표시, 원료, 알레르기, 영양표시, 기능성 광고, HACCP/GMP 리스크 저장
6. `MatchResult` 테이블: 후보 공장, 매칭 사유, 확인 필요 질문 저장

## 규제 게이트

MVP의 첫 버전은 규제 판정을 자동 확정하지 않는다. [[../platform/Regulatory_Screening_System|규제 스크리닝 시스템]]이 RED/YELLOW/GREEN 플래그와 공장 확인 질문을 생성하고, RED가 있으면 라벨·상세페이지 문구와 샘플 발주서를 보류한다.
