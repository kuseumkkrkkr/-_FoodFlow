# 바이브 쿠킹: LLM 기반 디지털 트윈 쿠킹

## 한 줄 정의

바이브 쿠킹은 사용자의 식품 아이디어를 LLM으로 레시피, 원재료 BOM, 공정 조건, 원가 범위, 표시·인증 체크리스트까지 구조화한 뒤 실제 공장 발주가 가능한 개발 사양서로 바꾸는 디지털 트윈 쿠킹 모듈이다.

## 왜 필요한가

[[../10_Core_Idea|AI 기반 식품 OEM/ODM 공장 매칭 플랫폼]]은 공장 매칭이 핵심이지만, 공장에 전달되는 입력이 부정확하면 매칭 품질이 떨어진다. "저당 고단백 쿠키를 만들고 싶다" 같은 문장은 제품 방향은 알 수 있어도 원재료, 배합비, 알레르기, 포장, 보관, MOQ, 원가 목표가 빠져 있어 실제 견적과 샘플 발주로 이어지기 어렵다.

바이브 쿠킹은 발주 전 단계에서 아이디어를 제조 가능한 언어로 바꾼다. 사용자가 맛, 식감, 가격대, 영양 목표, 판매 채널을 입력하면 LLM이 가상의 조리·제조 시뮬레이션을 수행하고, 공장 매칭에 필요한 조건을 정리한다.

## 핵심 산출물

| 산출물 | 설명 | 발주 연결 |
|---|---|---|
| 제품 컨셉 카드 | 타깃 고객, 섭취 상황, 판매 채널, 차별점 | B2B/B2C 상품기획 검토 |
| 레시피 초안 | 주원료, 부원료, 대체 가능 원료, 배합 방향 | 샘플 개발 요청 |
| 원재료 BOM | 원료명, 예상 투입량, 원산지/인증 요구, 알레르기 | 재료 발주와 원가 산출 |
| 공정 조건 | 배합, 성형, 가열, 냉각, 충진, 포장, 보관 | 공장 설비 매칭 |
| 규제 체크 | 영양표시, 알레르기, 고카페인, 당알코올, 원산지 | 라벨 검수와 리스크 관리 |
| 원가·MOQ 가정 | 목표 소비자가, 예상 제조원가, 테스트 수량 | 견적 요청과 수량 협상 |
| 검증 질문 | 맛, 물성, 보관 안정성, 생산성, 표시 리스크 | 샘플 테스트 설계 |

## 디지털 트윈 개발 흐름

1. 제품 의도 입력: 맛, 효능, 식감, 가격, 판매 채널, 타깃 고객을 자연어로 받는다.
2. 트렌드 매핑: [[../korea_trends/00_Korea_Trend_Map|한국 트렌드 맵]]의 저당, HMR, 단백질, 온라인/D2C, 케어푸드 조건과 연결한다.
3. 가상 레시피 생성: LLM이 원료 후보, 대체 원료, 배합 방향, 제조 난도를 제안한다.
4. BOM 구조화: 원재료를 주원료, 기능성 원료, 감미/향미, 안정화, 포장재로 나누고 발주 단위를 정리한다.
5. 공정 시뮬레이션: 배합, 가열, 냉동, 레토르트, 발효, 건조, 충진 같은 공정 후보를 추출한다.
6. 규제 스크리닝: [[Regulatory_Screening_System|한국 식품 규제 스크리닝]]으로 영양표시, 알레르기, 기능성 광고, HACCP/GMP, 첨가물, 포장재 리스크를 플래그 처리한다.
7. 공장 매칭 입력 생성: [[Matching_Logic|AI 매칭 로직]]에 넣을 제품 유형, 설비, MOQ, 인증, 보관, 포장 조건을 만든다.
8. 샘플 발주서 생성: 공장에 보낼 샘플 요청서와 원료 확인 질문을 만든다.

## B2B 적용

B2B에서는 바이브 쿠킹을 제품 개발 워크룸으로 사용한다. 브랜드, 유통사, 프랜차이즈는 여러 제품 가설을 빠르게 비교하고, 실제 공장 견적 전에 발주 조건을 좁힐 수 있다.

| B2B 상황 | 바이브 쿠킹 역할 |
|---|---|
| PB 상품 개발 | 목표 소비자가, 마진, MOQ, 납기 기준으로 BOM과 공정 후보 비교 |
| 프랜차이즈 메뉴 개발 | 매장 조리 난도, 중앙공급 가능성, 냉장/냉동 물류 조건 정리 |
| 수출 제품 개발 | 할랄, 비건, 글루텐프리, 다국어 라벨, 현지 표시 조건 점검 |
| 케어푸드 납품 | 연화식, 단백질 강화, 저작/삼킴 난이도, 시설 납품 조건 구조화 |
| HMR/RMR 개발 | 냉동, 레토르트, 즉석섭취, 1인분 소분, 온라인 배송 조건 비교 |

## B2C 적용

B2C에서는 바이브 쿠킹을 소량 제작 가이드로 사용한다. 예비 창업자, 크리에이터, 커뮤니티 운영자는 복잡한 식품 제조 지식 없이도 아이디어를 샘플 제작 가능한 형태로 바꿀 수 있다.

| B2C 상황 | 바이브 쿠킹 역할 |
|---|---|
| 공동구매 상품 | 예상 수요, 최소 제작 수량, 원가, 포장 옵션을 단순화 |
| 크리에이터 한정판 | 맛과 콘셉트를 유지하면서 생산 가능한 카테고리로 변환 |
| 테스트 판매 | 샘플 수량, 보관 방식, 표시 문구, 알레르기 리스크 점검 |
| 로컬 식품 개발 | 지역 원료, 원산지 증빙, 소량 가공 가능 공장 조건 정리 |
| 건강간식 개발 | 저당, 고단백, 식이섬유, 대체당, 영양표시 조건 정리 |

## 재료 발주 구조

재료 발주는 제품 아이디어가 아니라 BOM 기준으로 관리한다. 대량 처리와 DB 쿼리 단순화를 위해 원재료는 자유 텍스트만 저장하지 않고 매칭에 필요한 최소 필드를 고정한다.

| 필드 | 예시 |
|---|---|
| ingredient_role | 주원료, 감미료, 단백질원, 식이섬유, 향미, 안정제, 포장재 |
| ingredient_name | 현미, 분리대두단백, 알룰로스, 난소화성말토덱스트린 |
| required_claim | 저당, 고단백, 글루텐프리, 비건, 국내산 |
| allergen_flag | 우유, 대두, 밀, 계란, 견과류 |
| substitute_allowed | 가능, 제한 가능, 불가 |
| sourcing_condition | 원산지, 인증, 최소 발주 단위, 리드타임 |
| process_risk | 가열 안정성, 냉동 안정성, 점도, 수분활성, 산패 |

## LLM 역할과 제한

LLM은 개발 의사결정을 보조하지만 식품 안전성, 법적 표시, 기능성 효능, 실제 유통기한을 확정하지 않는다. 플랫폼은 LLM 결과를 그대로 발주하지 않고 공장, 원료사, 시험기관, 라벨 검수 담당자가 확인할 수 있는 체크리스트로 변환해야 한다.

## MVP 검증 질문

1. 사용자가 자연어로 입력한 제품 아이디어를 공장 견적 가능한 BOM과 공정 조건으로 바꿀 수 있는가?
2. B2B 발주자가 기존 상품기획서보다 빠르게 샘플 요청서를 만들 수 있는가?
3. B2C 사용자가 소량 제작 불가능한 아이디어를 가능한 카테고리로 수정할 수 있는가?
4. 공장은 바이브 쿠킹 산출물을 보고 불필요한 왕복 문의를 줄일 수 있는가?
5. 저당, HMR, 단백질, 케어푸드, 수출 제품 중 어떤 카테고리에서 샘플 성공률이 가장 높은가?
